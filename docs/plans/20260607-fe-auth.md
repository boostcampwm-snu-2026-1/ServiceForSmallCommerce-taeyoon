## 설계 결정

- 클라이언트 UI 상태(토큰/유저)는 Zustand + `persist`(localStorage) — 새로고침 유지. 서버 상태(TanStack Query)와 역할 분리(frontend.md).
- store 는 features/auth/store.ts (테스트 필수 대상). 페이지는 app/ 아래(테스트 필수 대상 아님이나 login 페이지 1건 추가).
- 페이지는 'use client'. login/register 성공 시 setAuth 후 `/dashboard` 로 router.push.
- 기존 `features/auth/api`(login/register)와 `types`(AuthResponse 등) 재사용.

## 테스트 전략

- `store.test.ts`(필수): setAuth 후 token/user 반영, logout 후 초기화, persist key 존재. 테스트 간 store/localStorage 리셋.
- `login-page.test.tsx`(선택): login api 모킹, 제출 시 setAuth 호출 + next/navigation router.push('/dashboard') 호출 검증.
