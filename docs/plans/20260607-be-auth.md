## 설계 결정

- `UserRepository`, `AuthService` 는 trait (Mock 주입 / 교체 가능성). 기본 구현 `PgUserRepository`, `StandardAuthService`.
- 비밀번호 해시: argon2 직접 사용 (단일 구현, 포트 불필요).
- JWT: jsonwebtoken 직접 사용. Claims = { sub: user_id, exp }. `verify_token`은 단위 3 인증 미들웨어에서 사용.
- `User` 도메인 모델은 `password_hash` 포함 (내부용). API 응답은 핸들러에서 별도 직렬화(해시 미노출).
- `Plan` enum: serde lowercase rename (`free`/`starter`/`pro`), 기본 `free`. 제한 로직 없음.

## 테스트 전략

- 단위: `auth_service.rs` 내 `MockUserRepository`(in-memory)로 register 중복/login 실패/토큰 생성·검증 검증.
- 단위: `user_repo.rs` 내 testcontainer Postgres로 create/find_by_email 라운드트립.
- 통합: `auth_test.rs` — register 201(토큰 발급), 중복 이메일 409/400, login 200, 잘못된 비번 401.
