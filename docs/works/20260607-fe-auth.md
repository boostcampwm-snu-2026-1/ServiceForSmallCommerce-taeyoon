## 작업 요약

프론트엔드 인증 토대를 구현했다: 영속 Zustand 스토어, TanStack Query Provider, 로그인/회원가입 페이지.

## 변경 내용

- `src/features/auth/store.ts` — `useAuthStore`(token/user, setAuth/logout, persist `auth-storage`) + `store.test.ts` 3개
- `src/shared/providers/QueryProvider.tsx` — 'use client' QueryClientProvider(useState 1회 생성)
- `app/layout.tsx` — QueryProvider 래핑, metadata(Coupang Review AI)·lang="ko" 갱신
- `app/(auth)/login/page.tsx`, `app/(auth)/register/page.tsx` — 폼 → api 호출 → setAuth → /dashboard 이동, 에러/로딩 처리
- `tests/unit/login-page.test.tsx` — login/router 모킹, 제출 흐름 검증

## 주요 결정과 이유

- 토큰/유저는 Zustand+persist(클라이언트 상태), 서버 상태는 TanStack Query로 역할 분리(frontend.md).
- 페이지 테스트는 vitest include(`tests/unit/**`) 규칙에 맞춰 배치.
- 기존 features/auth의 api/types 재사용(명명된 타입, 인라인 금지).
