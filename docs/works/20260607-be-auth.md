# BE 인증 (회원가입/로그인)

- **날짜**: 2026-06-07
- **플랜**: docs/plans/20260607-be-auth.md
- **로드맵**: docs/plans/20260607-feature-roadmap.md (단위 1/6)

## 작업 요약

JWT + argon2 기반 회원가입/로그인 API를 Hexagonal 구조로 구현했다. 수익(플랜 제한) 로직은 제외했고, `users.plan` 컬럼은 기본값 `free`로만 둔다.

## 변경 내용

### 1. 마이그레이션
- `backend/migrations/00002_users.sql` — users 테이블 (id, email UNIQUE, password_hash, plan, created_at)

### 2. 도메인
- `backend/src/domain/models/user.rs` — `Plan` enum(Free/Starter/Pro, serde lowercase, Default=Free, `from_db_str`/`as_str`), `User`(password_hash 직렬화 derive 없음)
- `backend/src/domain/ports/user_repository.rs` — `UserRepository` trait (create/find_by_email/find_by_id)

### 3. 어댑터
- `backend/src/adapters/postgres/user_repo.rs` — `PgUserRepository`. UNIQUE 위반(23505)→`BadRequest`. testcontainer 단위테스트 2개.

### 4. 서비스
- `backend/src/application/auth_service.rs` — `AuthService` trait + `StandardAuthService`(argon2 해시/검증, jsonwebtoken 발급/검증, register/login/verify_token). MockUserRepository 단위테스트 7개.

### 5. HTTP
- `backend/src/http/handlers/auth.rs` — register(201)/login(200), `UserView`(password_hash 비노출)/`AuthResponse`
- `backend/src/http/state.rs` — `auth_service: Arc<dyn AuthService>` 필드 + `new` 시그니처 확장
- `backend/src/http/router.rs` — `/api/v1/auth/register`, `/api/v1/auth/login`
- `backend/src/main.rs` — PgUserRepository → StandardAuthService → AppState 조립

### 6. 테스트
- `backend/tests/helpers/app.rs` — AppState 조립 동기화 + `register` 헬퍼
- `backend/tests/auth_test.rs` — 통합 테스트 5개(register 201, 중복 400, login 200, 잘못된 비번 401, 미가입 401)

## 주요 결정과 이유

- **핸들러 수동 작성**: `post_endpoint!` 매크로는 `Json<Resp>`만 반환 → register=201/login=200 상태코드와 중첩 `user` 객체를 표현 못 함. code-rules의 수동 작성 규칙(명시적 Result 타입, Serialize 구조체, json! 금지) 준수.
- **sqlx 런타임 쿼리(`query_as`+`FromRow`)**: CI는 빈 Postgres만 제공하고 컴파일 전 마이그레이션/오프라인 캐시가 없어 컴파일타임 매크로(`query_as!`)는 CI를 깨뜨림.
- **`Plan::from_db_str`**: clippy `should_implement_trait`가 `from_str`를 표준 `FromStr`와 혼동 → 명시적 이름 사용.
- **UserRepository / AuthService 를 trait 로**: Mock 주입 및 향후 교체 가능성 확보.
