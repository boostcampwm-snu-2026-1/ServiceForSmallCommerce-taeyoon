### 4. Unauthorized(401) 버그 수정 (task-with-harness)
- **원인**: `frontend/src/shared/api/client.ts`(`apiClient`)가 `Authorization: Bearer <token>` 헤더를 전혀 붙이지 않아, 인증이 필요한 모든 호출(분석 생성/조회/목록, `/users/me`)이 401
- curl로 확정: 토큰 없이 `POST /api/v1/analyses` → 401, 토큰 포함 → 202 Accepted (백엔드 정상)
- `apiClient`가 store의 토큰을 읽어 헤더에 싣도록 수정 + 회귀 방지 테스트 3개 추가 (37개 통과)
- 커밋: `fix: apiClient가 인증 토큰을 Authorization 헤더로 첨부`

## 주요 결정과 이유
- **모델로 `gemini-2.0-flash-lite` 확정**: 역대 최저가인 `gemini-1.5-flash-8b`는 1.5 세대 API 지원 종료(deprecated) 진행 중 → 호출 실패 위험으로 제외. 현재 지원 모델 중 최저가(~$0.075 input / $0.30 output per 1M tok)로 선택
- **메인화면은 옵션 1(리다이렉트) 채택**: 별도 랜딩 페이지 신규 구현 대신 인증 상태 기반 리다이렉트만으로 루트 보일러플레이트 노출 문제 해결 (사용자 선택)
- **401은 순수 프론트 버그로 확정**: 백엔드 `AuthUser` 추출기는 정상이고 토큰도 store에 저장돼 있었으나 클라이언트가 헤더에 싣지 않음 — curl로 양방향 검증 후 수정

## 발생한 문제와 해결
- **루트 `/` 접속 시 Next.js 기본 스타터 페이지 노출**: `app/page.tsx`가 보일러플레이트 그대로 → 실제 앱 화면은 라우트 그룹(`(auth)`/`(dashboard)`) 안에 존재함을 확인 → 인증 상태 기반 리다이렉트로 교체
- **분석 시작 시 Unauthorized(401)**: `apiClient`가 토큰을 헤더에 싣지 않던 것이 원인 → 토큰 첨부 로직 추가 + 회귀 테스트로 해결
- **라우트 페이지의 테스트 관례**: `tests/unit/`에 페이지별 테스트가 존재하는 패턴 발견 → 루트 페이지도 동일하게 리다이렉트 분기 테스트 추가하여 harness 정렬 (`User` 타입의 `plan` 필드까지 맞춤)
