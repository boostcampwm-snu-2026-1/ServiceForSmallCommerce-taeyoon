# 세션 정리 - 20260607

## 작업 목표
1. **LLM 교체**: AI 리뷰 분석기를 Anthropic Claude → Google Gemini로 교체 (API key 제공받음)
2. **모델 비용 최적화**: 기본 Gemini 모델을 가장 저렴한 모델로 변경
3. **로컬 환경 기동**: mock 없이 실제 어댑터로 앱을 직접 띄워서 사용자가 동작 확인

## 진행한 작업

### 1. LLM 교체 (Claude → Gemini)
`task-with-harness` 워크플로우 전 단계(컨텍스트 로드 → 계획 → 실행 → 검증 → 커밋)를 거쳐 진행.

- **모듈 rename**: `adapters/claude/` → `adapters/gemini/`, `ClaudeAiAnalyzer` → `GeminiAiAnalyzer`
- **API 연동 변경**: 엔드포인트 `models/{model}:generateContent`, 인증 헤더 `x-goog-api-key`, 요청 body `contents/parts` + `generationConfig.responseMimeType` 구조로 교체
- **포트 계약 유지**: `AiAnalyzer` trait과 `Insights` 도메인 모델을 그대로 유지 → 호출부(`StandardAnalysisService`) 무수정
- **import 사이트 일괄 수정**: `config.rs`, `main.rs`, `analysis_service.rs`(테스트 import), 통합 테스트 헬퍼
- **env/스펙 갱신**: `.env.example` 갱신, `.env`에 실제 키 저장(gitignore 확인), 스펙 파일 3개 + `harness.md`의 "Claude API" 참조 갱신
- **검증**: `cargo check` 통과, 단위 테스트 33개 + 통합 테스트 10개 통과
- 커밋: `a679eb8`

### 2. 모델 비용 최적화 (gemini-2.5-flash → gemini-2.0-flash-lite)
- 사용자에게 "어떤 최저가 모델인지" 확인 → `gemini-2.0-flash-lite` 확정
- `DEFAULT_MODEL` 상수, `config.rs` fallback 문자열/주석, env, 스펙, endpoint 테스트의 모델 문자열까지 일괄 교체
- 전체 테스트 43개 통과
- 커밋: `a84d9d6`

### 3. 로컬 기동 (실제 어댑터)
- PostgreSQL(docker compose) 기동, 부팅 시 `sqlx::migrate!`로 마이그레이션 자동 실행 확인
- 프론트 env 생성, 백엔드(8080)·프론트엔드(3000) 백그라운드 기동
- 실제 어댑터 확인: `GeminiAiAnalyzer`(실제 Gemini 호출) + `HttpCoupangCrawler`(실제 크롤러), mock 미사용
- 프론트 `/api/v1/*` ↔ 백엔드 라우트 prefix 일치 확인
- health 체크 통과 → 모두 정상 기동

## 주요 결정과 이유
- **포트(trait) 계약 유지**: `AiAnalyzer`/`Insights`를 그대로 두어 LLM 공급자만 갈아끼우고 호출부는 건드리지 않음 → 교체 가능성 우선 설계 원칙 적용
- **`gemini-2.0-flash-lite` 선택**: 역대 최저가인 `gemini-1.5-flash-8b`는 1.5 세대 API 지원 종료(deprecated) 진행 중이라 호출 실패 위험으로 제외. 현재 지원 모델 중 최저가(~$0.075 input / $0.30 output per 1M tok) 선택. `GEMINI_MODEL` env로 override 가능하게 유지
- **mock 미사용 기동**: 사용자가 직접 동작 확인을 원했으므로 실제 Gemini + 실제 크롤러로 구동
- **`.env`에 실제 키 저장**: 양쪽 레벨 모두 gitignore 처리되어 있음을 확인 후 커밋에서 제외

## 발생한 문제와 해결
- **중간 컴파일 에러**: 모듈 rename 중 import 사이트들이 일시적으로 깨짐 → atomic refactor 특성상 예상된 상태. 모든 import 사이트 갱신으로 해결
- **통합 테스트 헬퍼 누락**: `cargo check`는 테스트를 컴파일하지 않아 통과했지만, 통합 테스트 헬퍼가 여전히 구 모듈을 참조 → `cargo test` 단계에서 발견·수정
- **루트 `/` 접속 시 Next.js 기본 스타터 페이지 노출**: `app/page.tsx`가 기본 템플릿 그대로 남아있어서 발생. 실제 앱 페이지는 `(auth)`/`(dashboard)` 라우트 그룹 내에 존재 → **세션 종료 시점에 실제 라우트 경로 확인 진행 중이었음 (미완)**
