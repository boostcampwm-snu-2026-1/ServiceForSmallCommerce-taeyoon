## 진행한 작업

### 1. Claude → Gemini 교체 (task-with-harness 워크플로우)
- `adapters/claude/` → `adapters/gemini/` 모듈 rename, `ClaudeAiAnalyzer` → `GeminiAiAnalyzer`
- API 연동 변경: 엔드포인트 `models/{model}:generateContent`, 인증 헤더 `x-goog-api-key`, body `contents/parts` + `generationConfig.responseMimeType`
- `AiAnalyzer` 포트 계약과 `Insights` 도메인 모델은 그대로 유지 → 호출부(`StandardAnalysisService`) 무수정
- 영향 파일 15개: 코드(analyzer/config/main/adapters mod), 테스트 import, env(`.env.example` + 실제 키 `.env`), 스펙 3종(harness 포함)
- 검증: `cargo check` + 33개 lib 단위 테스트 + 10개 통합 테스트 통과 → 커밋

### 2. 기본 모델 → gemini-2.0-flash-lite
- `DEFAULT_MODEL` 상수, `config.rs` fallback, env, 스펙, endpoint 테스트까지 모델 문자열 일괄 교체 (8개 파일)
- `GEMINI_MODEL` env로 override 가능
- 43개 테스트 전체 통과 → 커밋

### 3. 로컬 환경 기동 (mock 미사용)
- PostgreSQL(docker compose) 기동, 프론트 env 생성, 마이그레이션은 부팅 시 `sqlx::migrate!` 자동 실행
- 백엔드(8080) + 프론트엔드(3000) 백그라운드 기동, health/경로 정합성 확인
- 실제 어댑터 확인: AI = `GeminiAiAnalyzer`(real, 키 설정됨), 크롤러 = `HttpCoupangCrawler`(real)

## 주요 결정과 이유
- **포트 계약 유지로 호출부 무수정**: `AiAnalyzer` trait과 `Insights`를 그대로 둬서 LLM 제공자 교체가 어댑터 레이어에 국한됨 (교체 가능성 우선 설계)
- **모델로 `gemini-2.0-flash-lite` 선택**: 역대 최저가인 `gemini-1.5-flash-8b`는 1.5 세대 API 지원 종료(deprecated) 진행 중 → 호출 실패 위험으로 제외. 현재 지원 모델 중 최저가(~$0.075 input / $0.30 output per 1M tok)인 2.0-flash-lite 확정
- **모듈 rename은 원자적 리팩터로 단일 처리**: import 사이트가 동시에 깨지므로 서브에이전트 병렬 분해 대신 메인이 일괄 수정
- **`.env`에 실제 키 저장**: 양쪽 레벨 모두 gitignore 처리되어 커밋에서 안전하게 제외

## 발생한 문제와 해결
- **테스트 import 누락**: `cargo check`는 테스트를 컴파일하지 않아 통과했으나, `analysis_service.rs` 및 통합 테스트 헬퍼가 옛 모듈을 참조 → 발견 후 import 수정하여 테스트까지 통과
- **루트 `/` 접속 시 Next.js 기본 스타터 페이지 노출**: `app/page.tsx`가 보일러플레이트 그대로였음. 실제 앱 화면은 라우트 그룹(`(auth)`/`(dashboard)`) 안에 존재 → 실제 경로 안내(`/register`, `/login`, `/dashboard`, `/analyses/[id]`)
